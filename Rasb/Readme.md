Описание проекта:
	Система для мониторинга и валидации показаний датчиков температуры, освещенности и давления в режиме реального времени.



Архитектура системы: 
	Сервер (Rasb_srv.py) - Принимает данные контракта, полученного с помощью скрипта script_download.sh, обрабатывает JSON формат, отправляет их на микроконтроллер. Затем принимает показания датчиков с микроконтроллера и проверяет их в соответствие заданным диапазоном ;
	Скрипт загрузки (script_download.sh) - Получает контракт c плотформы confident и запускает сервер ;
	Тестовый скрипт (test_script_srv.sh) - Имитирует отправку данных с микроконтроллера;



Запуск системы: 
	#Перед запуском убедиться, что контракт располагается в нодах ;
	# 1. Запустить скрипт загрузки	./script_download.sh ;
	# 2. В другом терминале запустите тестовый скрипт для отправки данных 	./test_script_srv.sh ;
	# 3. Для завершения работы создайте файл FLAG_FILE.txt (Он иммитирует точку останова, которая может быть абсолютно людей, в зависимоти от поставленной задачи) ;



Описание файлов:
	Rasb_srv.py - Основной сервер ;
	Назначение: Сервер мониторинга датчиков ;
	Функциональность: Принимает JSON-контракт через stdin с диапазонами датчиков
	Слушает порт 55555 для получения данных с датчиков
	Накопливает показания датчиков
	При обнаружении FLAG_FILE.txt вычисляет средние значения
	Проверяет соответствие средних значений заданным диапазонам
	Выводит результаты валидации

	ПРИМЕР ФОМАТА ВХОДНЫХ ДАННЫХ ОТ КОНТРАКТА :
		{
  		 "contract_id": [{
      			"type": "string",
      			"value": "20/35",  // диапазон температуры
      			"key": "temperature"},
    			{
      			"type": "string", 
      			"value": "700/1100",  // диапазон освещенности
      			"key": "light"},
   			{
      			"type": "string",
      			"value": "900/1100",  // диапазон давления
      			"key": "press"
    			}]
		}
	ПРИМЕР ВХОДНЫХ ДАННЫХ ОТ ДАТЧИКОВ:
		({temperature:25},{light:450},{press:980})

	script_download.sh - Скрипт загрузки c ноды ;
	Назначение: Получение контракта и запуск сервера ;
	Шаги выполнения:
	Получает ID контракта с API сервера
	Загружает данные контракта
	Передает контракт в Python сервер через stdin
	Автоматически удаляет FLAG_FILE.txt при завершении
	Зависимости: curl для HTTP запросов и python3 для запуска сервера


	test_script_srv.sh - Тестовый скрипт;
	Назначение: Имитация работы датчиков
	Порт сервера: Сервер слушает на порту: 55555
	Изменить можно в строке: port = 55555
	Диапазоны датчиков по умолчанию: Температура: 20-35°C Освещенность: 700-1100 люмен Давление: 900-1100 гПа
